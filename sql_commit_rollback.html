SQL COMMIT and ROLLBACK

In SQL, the COMMIT and ROLLBACK statements are used to manage transactions. Transactions ensure the integrity of data by grouping one or more SQL statements into a single unit of work.

What is a Transaction?

A transaction is a sequence of SQL statements that are executed as a single unit. If all statements succeed, the transaction is committed. If there is any error or if the user decides to undo the changes, the transaction can be rolled back.

  • COMMIT: Saves all changes made by the transaction permanently.
  • ROLLBACK: Reverts all changes made by the transaction.

COMMIT Syntax

COMMIT;

The COMMIT command is used to save all changes made during the current transaction.

ROLLBACK Syntax

ROLLBACK;

The ROLLBACK command is used to undo transactions that have not yet been saved to the database.

Example Using COMMIT

Suppose we are inserting a new record into a table and want to ensure that the change is permanent:

START TRANSACTION; INSERT INTO Employees (ID, Name, Department) VALUES (5, 'Alice', 'IT'); COMMIT;

This transaction inserts a new employee and then commits the change, making it permanent in the database.

Example Using ROLLBACK

Suppose we realize there's an error after making a change. We can undo it:

START TRANSACTION; UPDATE Employees SET Department = 'Finance' WHERE ID = 5; ROLLBACK;

This transaction updates the department but then rolls it back, so the change is discarded.

Important Notes

  • Some databases (like MySQL) may auto-commit each statement by default. Use START TRANSACTION or set autocommit to OFF to use transactions explicitly.
  • Transactions are useful when performing multiple operations that must either all succeed or all fail together.
  • COMMIT makes changes permanent, ROLLBACK undoes them.
